# Contributing to Distilled CS

Distilled CS is an open, community-driven Customer Success framework. Contributions are welcome — refinements to the framework content, new playbooks, improved metric definitions, bug fixes in the code, and documentation improvements all help the project grow.

## Where to start

- **Framework content** (assessment questions, playbooks, metrics, principles, glossary): edit `src/data/lean-cs-data.json`. This is the single source of truth for framework content.
- **Articles and long-form content**: add Markdown files to `src/content/blog/`.
- **Site code** (Astro components, scripts, styling): standard pull request workflow.
- **Voice and tone**: the project uses an advisory, peer-to-peer tone rather than a marketing one. Avoid superlatives, urgency hooks, and hollow provocations. Prefer concrete advice over abstract claims. Match the register of existing copy on adjacent pages before writing new copy.

## How to contribute

1. Open an issue or comment on an existing one to discuss larger changes before starting work.
2. Fork the repository and create a feature branch.
3. Make your changes. For UI changes, please test in both light and dark mode.
4. Open a pull request with a clear description of what you changed and why.

For small typo fixes or copy improvements, you can open a pull request directly without an issue.

## Code of conduct

Be kind and constructive. Disagree on ideas, not on people. Assume good intent. If a discussion gets stuck, step away and come back later.
